# GBot Bubble Agent 商业资产系统设计规范 (V1)

本规范定义了 GBot Miniapp 首页 Bubble Agent 形象重构后的商业资产体系。该系统采用“**应用内轻量资产 + 用户自愿铸造链上 Passport**”的双层模型，在符合各端合规要求的前提下，为泡泡召唤兽赋予长期的运营与收藏价值。

---

## 1. 资产双层模型概述

GBot Bubble Agent 资产分为两层：
1. **App 内 Agent 资产（默认状态）**：储存于游戏中心化数据库，官方默认出售/发放的均为此类资产。支持养成、装备技能、配饰装扮、展示唯一编号等，但不具备链上属性，不可直接链上交易。
2. **链上 Agent Passport（用户主动铸造）**：用户可在任意时间自愿选择将 App 内资产“铸造（Mint）”到区块链上。铸造成功后生成具有唯一 `token_id` 的 NFT，支持在外部 NFT 市场中展示、赠予、挂单与交易。

---

## 2. App 内 Agent 资产数据结构

App 数据库中每个 Agent 资产包含以下核心字段：

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `agent_id` | String (UUID) | 游戏内全局唯一资产 ID，不可更改。 |
| `display_no` | String | 展示编号（如 `GBOT-000888`），遵循泡泡编号体系。 |
| `owner_user_id`| String | 当前归属的 GBot 用户 ID。 |
| `bound_wallet` | String | 用户绑定且经签名确认的区块链钱包地址。 |
| `series` | String | 资产系列（如 `Genesis` / `Season-1` 等）。 |
| `color_gene` | String | 颜色谱系（如 `BlackGold` / `SmokeGray` 等）。 |
| `rarity` | String | 稀有度等级（`Common` / `Rare` / `Epic` / `Genesis`）。 |
| `level` | Integer | 泡泡当前等级（影响日常行动力上限等游戏属性）。 |
| `skills` | Array[String] | 已解锁和装配的任务技能。 |
| `cosmetics` | Array[String] | 已解锁和装配的配饰/特效 ID。 |
| `created_at` | DateTime | 资产生成时间。 |
| `mint_status` | Enum | `unminted` (未铸造) \| `minting` (铸造中) \| `minted` (已铸造) \| `failed` (铸造失败)。 |

## 3. 泡泡编号体系 (Numbering Rules)

每个泡泡资产在 App 内生成时，即分配一个**游戏内唯一编号（display_no）**。

### 3.1 编号规则与命名空间
- **创世编号**：`GBOT-GENESIS-[001-999]`（限量 999 只，用于早期核心贡献者与限量空投）。
- **赛季编号**：`GBOT-S[赛季号]-[五位顺序号]`，例如 `GBOT-S1-03456`。
- **普通编号**：`GBOT-[六位顺序号]`，例如 `GBOT-000888`。
- **稀有尾号**：尾号为 `666`, `888`, `999` 或豹子号 `111111` 等，在游戏内展示时带有特殊铭牌边框。
- **公会号段**：预留给特定合作伙伴或社区公会的专属号段，如 `GBOT-GUILD-[0001-1000]`。

### 3.2 资产映射关系
- **区别**：`agent_id` 是系统内部的 UUID，用于逻辑计算与防伪；`display_no` 是面向用户的展示编号。
- **链上映射**：当用户选择 mint 时，链上智能合约的 `token_id` 将通过数学算法与 `display_no` 进行一对一的硬编码锁定（如 `token_id = hash(display_no)` 或直接使用 `uint256` 形式的编号），确保链上所有权与游戏内资产的一致性。

## 4. 泡泡颜色谱系 (Color Pedigree)

新版“冷脸泥泡泡”打破单调的换色，将其定义为具有不同性格特质的商业系列：

| 谱系名称 | 视觉特征 | 商业定位与收藏价值 |
| :--- | :--- | :--- |
| **黑金泥泡泡** | 主线黑金渐变，金砂流动 | **核心主线**。代表冷静、专注，是 GBot 的经典商务看板形象。 |
| **烟灰泥泡泡** | 灰度半透明，烟熏雾气 | **Meme 搞怪**。代表慵懒、摆烂（Idle 特效幅度极小）。 |
| **暗紫泥泡泡** | 幽邃暗紫核心，电光微粒 | **策略偏好**。代表神秘、精算（战报文案偏逻辑分析）。 |
| **冰蓝泥泡泡** | 水流浅蓝透明，核心泛冷光 | **数据流**。代表理性、客观（在战报中偏向数据表格展示）。 |
| **赤金泥泡泡** | 熔岩红色渐变，流沙涌动 | **高能偏向**。代表进取、进攻（战斗行动特效偏火花）。 |
| **绿毒泥泡泡** | 毒液荧光绿，酸性气泡 | **风险追逐**。代表高频行动（战报语气偏赌徒 meme 趣味）。 |
| **白银泥泡泡** | 液态水银质感，金属反光 | **早期纪念外观**。作为 Genesis 纪念系列。 |

> [!WARNING]
> **合规约束**：颜色与谱系划分**仅影响**资产外观、收藏稀有度、日志战报生成的趣味语气、以及特有技能特效。**绝对禁止任何“稀有颜色赚取更多代币/金币”或“高级皮肤收益更高”等金融化回报承诺**。

## 5. 配饰槽位与装扮系统

为了让泥泡泡具备更强的个性表达与商业可延展性，提供以下 8 个独立的配饰/特效槽位：
1. **表情包**：如“冷漠脸”、“无语翻白眼”、“嫌弃”等不同死鱼眼贴图。
2. **头饰**：小礼帽、天线、发夹等。
3. **面饰**：墨镜、防毒面具、半遮面面罩。
4. **身体特效**：浮动微粒、核心燃烧火焰、环绕金色碎屑等。
5. **底座**：科技感托盘、泥浆滩、黑金齿轮。
6. **背景**：深色玻璃、渐变黑金粒子光斑。
7. **动作包**：特殊的 tap 压扁变形物理常数参数、特别的 reward 庆祝动作。
8. **铭牌**：展示唯一编号时的个性化花纹边框。

> [!NOTE]
> 所有配饰均为纯外观、社交展示和自我表达工具，不与任何代币获取速率挂钩。

## 6. 用户自愿铸造 Agent Passport 流程

链上 Passport **不是自动生成**的，需由用户主动发起并支付 Gas 费用。

### 6.1 链上数据结构
当 mint 成功后，链上合约将存储并锁定以下元数据：
- `chain`: 部署的目标链（如 ETH / Layer 2）。
- `contract_address`: GBot Agent Passport 合约地址。
- `token_id`: 链上唯一 Token ID。
- `agent_id`: 映射的游戏内 UUID。
- `display_no`: 映射的游戏内展示编号。
- `owner_wallet`: 当前持有该 NFT 的钱包地址。
- `metadata_uri`: 指向去中心化存储（IPFS）的 Metadata JSON（包含角色图片、颜色、稀有度、等级等快照）。
- `minted_at`: 铸造时间戳。
- `transfer_status`: `idle | transferring | frozen`。
- `marketplace_status`: `listed | unlisted`。

## 7. Mint 状态迁移与 UI 文案

App 页面中必须清晰展示泡泡的 Mint 状态，并使用准确的合规文案引导用户：

| 状态 (mint_status) | 游戏内显示文字 | 行为约束 |
| :--- | :--- | :--- |
| **`unminted`** | `游戏内资产 · 可选择铸造 Agent Passport` | 允许养成、升级、执行任务；不可进行链上转赠或交易。 |
| **`minting`** | `等待钱包签名/确认中...` | 暂时冻结资产升级与配饰更换，直至链上交易确认或失败。 |
| **`minted`** | `已铸造链上 Passport (Token ID: #xxxx)` | 解锁链上转赠、第三方市场交易；游戏内仍可正常使用。 |
| **`failed`** | `铸造失败，点击重试` | 恢复为 unminted 状态，允许用户再次发起铸造。 |

---

## 8. 交易边界与钱包归属同步

### 8.1 交易权限与限制
- **未铸造（unminted）**：无法进行链上转让与交易。
- **已铸造（minted）**：允许在支持 ERC-721/1155 的第三方市场（如 OpenSea）挂单交易。

### 8.2 归属同步机制
- 当用户在链上将 Passport NFT 转让或售出给他人后，GBot 系统将通过 Web3 Event 监听器自动检测到 `Transfer` 事件。
- 系统通常会在链上确认和索引完成后同步归属状态（同步时间取决于目标链确认速度、索引服务与钱包绑定状态，多数情况下可在数十秒到数分钟内完成）：
  1. 若新持有者已在 GBot App 内绑定钱包，系统将更新 App 内展示归属为新持有者。
  2. 若新持有者未注册/绑定，该泡泡将进入待认领状态，直到对应钱包所有者登录 App 并绑定声明归属。
  3. 同步完成后，原持有者将不再显示为该 Agent 的当前持有人，App 会根据链上最终归属更新使用权限，存在链上确认中的短暂同步窗口。

## 9. 合规文案红线 (Compliance Copywriting Guidelines)

为了规避潜在的金融合规风险，所有产品界面、广告文案、说明文档必须遵守以下文字规范：

### 🚫 禁用词与表达 (Prohibited)
*   **禁止描述自动上链**：“官方上链”、“所有泡泡都是 NFT”、“购买即自动获得可交易 NFT”。
*   **禁止承诺升值**：“泡泡一定能升值”、“购买低编号能赚取超额溢价”、“早期泡泡具备理财升值属性”。
*   **禁止收益挂钩**：“稀有颜色赚得更多”、“购买稀有泡泡保证回报”、“高级泡泡收益更高”。
*   **禁止承诺回报**：不得使用收益承诺、固定式回款、确定性盈利、零风险投资等表达。

###  推荐词与表达 (Recommended)
*   **描述唯一性**：“游戏内唯一编号”。
*   **描述自愿上链**：“可选择铸造为链上 Agent Passport”。
*   **描述链上功能**：“铸造后支持链上展示、转让与交易”。
*   **描述游戏内价值**：“未铸造泡泡仍可在 GBot 内养成、升级、装备技能和执行任务”。
*   **描述用户行为**：“链上操作与交易由用户钱包自行确认”。
*   **描述配饰**：“配饰仅用于个性化展示与社交表达，不代表收益承诺”。

### ⚠️ 资产身份稳定性红线 (Asset Identity Stability Red Lines)
*   **编号永久固定**：资产编号 `display_no` 必须在生成时由 UUID 稳定哈希决定，**绝对禁止**随等级升级而发生编号变更。
*   **稀有度与等级解耦**：资产稀有度 `rarity` 由天赋 Hash 决定，**绝对禁止**与角色等级（Level）挂钩或随等级提高而自动“进化”稀有度等级。
*   **Genesis 身份恒定**：创世纪念系列等特定系列属性，必须在生成时即硬编码绑定，**绝对禁止**将后期升级行为作为获得创世身份的手段，以防违背资产稀有度公平性与代币化合规要求。
