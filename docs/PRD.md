# GrowthBot 产品需求文档

## 1. 产品概要

GrowthBot 是 Telegram 原生的全球任务赏金执行网络 (Telegram-native Global Bounty Execution Network)。项目方通过存入 TON/GRAM/USDT 等生态资产发布赏金任务，Agent（由具备特定执行能力的技能卡 NFT 驱动）负责发现、接取、拆解、执行或引导用户完成任务，平台通过链接、链上记录和人工审核等方式验收，最终结算奖励并沉淀用户与 Agent 的信誉。

> [!IMPORTANT]
> **旧系统兼容原则**：所有新文档与规划设计不得要求立即替换当前 V0 Earn / Task / Marketplace / Admin 流程；Bounty Network 是当前任务系统的战略升级方向，不是立即推翻现有实现。在 Phase 0 阶段，系统仅承担任务发现、任务整理、链接提交和战报反馈，不新增真实钱包结算与 Agentic Wallet 主网交易。

## 2. 目标用户

主要用户：
- Web3 社区用户和空投任务参与者。
- TON 生态探索者。
- Telegram 群、项目社区和增长型用户。

次要用户：
- 需要真实社区增长、社交传播、测试用户和任务分发的项目方。
- 未来可投放赏金任务的 KOL、社区和合作方。

## 3. 核心承诺

培养你的 Agent，让它学习技能、发现任务、整理步骤、辅助提交验收，并通过积分/生态资产积累未来奖励资格和分配权重。

GrowthBot 不承诺固定收益、固定兑换、保证空投或无风险回报。

## 4. 核心循环

1. 用户打开 Telegram Bot。
2. 用户免费领取基础 Agent。
3. 用户打开启动技能包，获得基础 Agent 技能卡。
4. Agent 展示可参与任务，包括社交关注、项目交互或链上打卡。
5. 用户点击任务，Agent 打开任务引导浮层，展示任务价值、步骤和外部直达链接。
6. 用户或 Agent（在授权且安全的情况下）完成任务执行。
7. 提交任务证明（链接、交易哈希或 API 自动校验）。
8. 平台按任务类型做格式或链上事件校验，审核通过后发放奖励。
9. 用户可把未学习、未绑定的技能卡上架市场，或购买其他技能卡培养 Agent。
10. 已学习或已装备的技能卡绑定 Agent，默认不可交易。

## 5. 产品支柱

### 5.1 Agent 技能学习

原有“盲盒 / 道具 / 装备”的公开叙事升级为：
- 技能学习包 / 技能包。
- Agent 技能卡。
- Agent 技能学习 / 装备。

技能卡不是普通加成道具，而是 Agent 的能力资产。技能类别包括：
- 任务发现类。
- 任务整理类。
- 验收与信誉类。
- 增长传播类。
- 交易准备类。

### 5.2 任务分层与执行模式

根据智能体与用户的协同深度，赏金任务划分为以下四类：

| 任务类型 | 执行方式说明 | 是否需要用户最后一步确认 |
| :--- | :--- | :--- |
| **Human-final-step** | Agent 自动扫描并整理步骤，生成直达链接；用户手动完成登录、关注、加群等社交操作并提交链接凭证。 | **是** |
| **Agent-assisted** | Agent 辅助生成推特文案、预填表单、自动提醒截止时间、自动收集并提交任务证明凭证。 | **半自动** (需要授权) |
| **Agentic on-chain** | 由非托管的 Agentic Wallet（测试网 Beta 阶段）根据白名单模板，自动签名执行链上打卡、领水、Raffle 报名或 Badge Mint。 | **否** (预存小额Gas，自动执行) |
| **Project API task** | 项目方系统通过 API/Webhook 接口主动向平台回调，验收用户或 Agent 的具体执行结果。 | 视具体任务而定 |

### 5.3 唯一编号资产

每张技能卡都有唯一编号，例如 `skill_card_000001`。编号用于市场展示、运营追踪、审计和未来资产迁移预留。

### 5.4 脱敏与加密

技能卡内部效果、权重、公式和掉落参数不向普通用户明文展示。前端只展示：
- 技能名称摘要。
- 类别。
- 稀有度。
- 编号。
- 所属系列。
- 用途摘要。

### 5.5 任务链接验收

任务采用双阶段流程：
1. 用户提交完成链接，状态为 `submitted`。
2. 用户触发验收，状态短暂进入 `verifying`。
3. 链接格式符合任务平台规则后变为 `approved` 并发放积分。
4. 不符合规则则变为 `rejected` 并返回反馈。

高奖励任务和异常提交应进入后台人工审核或抽检。

## 6. 核心模块

### 6.1 Mini App

- Agent 首页：展示 Agent 等级、能量、积分和已学习技能。
- 任务页：展示可参与任务，并打开 Agent 任务引导浮层。
- 背包：展示技能包、技能卡、编号、系列和交易状态。
- 市场：买卖未学习、可转让的技能卡。
- 战队和排行：展示协作进度、战队解锁和用户排名。

### 6.2 Admin 后台

- 总览：展示用户、任务、市场和风险指标。
- 技能学习路径配置：管理技能包和掉落池。
- 技能卡统计：查看未学习、已装备、挂售中、已消耗和已过期数量。
- 任务链接验收：查看提交链接，支持手动通过或拒绝。
- 市场监控：查看挂单、成交和异常交易。
- 审计：记录危险操作和人工审核。

## 7. V0/V1 范围

- Telegram Mini App。
- Cloudflare Worker + D1 + Pages。
- Off-chain 积分和 POINT_TEST 内测市场。
- 技能卡唯一编号和脱敏展示。
- 链接型任务提交与基础验收。
- 后台人工审核入口。

V0 不接真实 TON 自动交易，不控制用户主钱包，不承诺积分和代币固定兑换。

## 8. 成功指标

- 新用户到 Agent 领取转化率。
- Agent 领取到启动技能包开启率。
- 任务提交数和验收通过率。
- 人均有效任务完成数。
- 技能卡挂单量、成交量和流动性。
- 邀请用户激活率。
- 后台人工审核耗时和异常提交率。
