# GrowthBot Economy

## 1. Economy Goal

GrowthBot's economy should create three feelings:

- Users feel they are building future opportunity eligibility.
- Users feel they can improve their Mission efficiency.
- Users feel the market is moving without them.

The economy must avoid promising fixed token returns.

## 2. Asset Model

### 2.1 Non-Tradable Assets

These assets protect fairness and anti-sybil quality:

- User Score.
- Soulbound contribution badges.
- Verified user status.
- Agent Mission history.
- Long-term activity level.

They should not be transferable.

### 2.2 Tradable Assets

These assets create market activity:

- Unopened boxes.
- Agent ability cards.
- Energy packs.
- Auto-run passes.
- Project tickets.
- Group boost cards.
- Strategy modules.

### 2.3 Restricted Assets

These may become tradable later, but should be controlled:

- Claim Credits.
- Launch allocation vouchers.
- Airdrop redemption tickets.

Recommended restrictions:

- Cooldown period.
- Project-specific transfer rules.
- Anti-sybil checks.
- Expiry.
- Marketplace fee.

## 3. Points System

### 3.1 Pending Points

Pending Points are produced by Agent activity.

Sources:

- Task completion.
- Daily Missions.
- Group mining.
- Referral quality.
- Project campaigns.
- Ability boosts.

Pending Points are not final. They must pass settlement.

### 3.2 User Score

User Score is long-term reputation and contribution weight.

It should be:

- Non-transferable.
- Anti-sybil adjusted.
- Used for ranking.
- Used for airdrop weighting.
- Used for eligibility.

### 3.3 Claim Credits

Claim Credits are the user's redeemable allocation units after settlement.

Uses:

- Join project airdrop pools.
- Redeem whitelist chances.
- Enter launch access pools.
- Buy selected ecosystem rights.

Important:

- Do not define a fixed token conversion rate too early.
- Each project may define its own redemption rules.

## 4. Energy System

Energy controls Agent work capacity.

Energy can be earned by:

- Daily login.
- Inviting verified users.
- Group mining participation.
- Holding abilities.
- Opening boxes.
- Subscription.

Energy is consumed by:

- Auto-running tasks.
- Entering project pools.
- Claiming rewards.
- Activating boosts.
- Joining raffles.

Energy is useful because it gives users daily decisions:

- Save energy.
- Spend energy on high-yield tasks.
- Buy or trade more energy.
- Join a group pool to get cheaper boosts.

## 5. Blind Box Economy

### 5.1 Box Types

Starter Box:

- Free.
- Gives new users basic rewards.
- Cannot create too much value.

Alpha Box:

- Daily or weekly limited.
- Contains stronger abilities.
- Designed for urgency.

Project Box:

- Issued by ecosystem projects.
- Contains project boosts, tickets, badges, and raffle entries.

Group Box:

- Unlocked by group participation.
- Better rewards when more verified users join.

Genesis Box:

- Scarce.
- Contains rare abilities and identity assets.

### 5.2 Drop Contents

Possible contents:

- Points Boost.
- Energy.
- Mission Pass.
- Launch Sniper Access.
- Project Priority Pass.
- Raffle Auto Entry.
- Group Mining Boost.
- Rank Shield.
- Claim Credit Converter.
- Genesis Badge.

### 5.3 Drop Table Policy

Drop rates should be visible enough to build trust.

For paid boxes:

- Show rarity tiers.
- Show expected categories.
- Avoid guaranteed token return language.
- Avoid "profit guaranteed" language.

## 6. Ability System

Ability properties:

- Name.
- Rarity.
- Duration.
- Uses remaining.
- Applicable projects.
- Transferable or soulbound.
- Stackable or non-stackable.
- Marketplace eligibility.

Examples:

- `2x Points Boost`: doubles Pending Points for selected tasks.
- `7-Day Mission Pass`: expands Mission access for 7 days.
- `Launch Sniper`: unlocks early project task pools.
- `Gas Saver`: reduces user-paid execution cost.
- `Rank Shield`: protects ranking tier for a short period.
- `High-Yield Scanner`: reveals higher-yield tasks.

## 7. Marketplace

Marketplace creates price discovery and asset liquidity.

MVP:

- Fixed-price listings.
- Buy and sell unopened boxes.
- Buy and sell transferable abilities.
- Basic floor price and volume stats.

Future:

- Rentals.
- Auctions.
- Bundles.
- Offers.
- Whale alerts.
- Group trading rooms.

Marketplace discovery surfaces:

- 24h volume.
- Floor price changes.
- Rare ability sales.
- Top holders.
- Top Agent inventory.
- Expiring abilities.
- New project box launch countdown.

## 8. Revenue Model

Revenue sources:

- Agentic Wallet activation fee.
- Agent subscription.
- Box sales.
- Marketplace trading fees.
- Ability rental fees.
- Project campaign setup fee.
- Project box issuance fee.
- Project task pool performance fee.

Recommended initial fee structure:

- Marketplace fee: 2%-5%.
- Box primary sale fee: project-specific.
- Subscription: 7-day and 30-day plans.
- Agent activation: includes deployment/service/gas buffer.

## 9. Anti-Sybil and Abuse Controls

Controls:

- Telegram account age signals.
- Group participation quality.
- Referral quality.
- Wallet uniqueness.
- Device/session risk.
- Activity timing patterns.
- On-chain behavior.
- Cooling period before Claim Credits.

High-risk behavior should reduce settlement rate from Pending Points to Claim Credits.
