## Card Rules

- Cards are public capability assets, not internal code modules.
- Cards can be owned, equipped, listed, or retired depending on their lifecycle rules.
- A card that is equipped should not be listed at the same time.
- Soulbound or starter-only cards may exist, but that must be explicit per card.
- Any new public card requires a new versioned canon; the 31-card set does not expand silently.
